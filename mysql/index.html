3.1处理百万级以上数据如何提高数据查询速度方法？

  1. 避免索引失效导致全表扫描的操作符
    • 操作符 != 或者 <>,推荐转换换成 =;
    • 条件语句column is null，推荐字段设置默认值，默认不能为null；
    • 操作符or，推荐换成 union all写法；
    • like '%search_test%'；
    • not in，推荐使用not exist写法；
    • 在where子句中对字段进行表达式操作/函数操作，如where id/2=100,可以改写成where id = 100*2；
  2. in写法，在in有界条件中，可以用in，但是in select id from table,这种最好通过exits或者innjoin写法；
  3. 索引字段遵循前缀匹配原则；
  4. 索引尽量选择数字型字段，如使用字符型会降低查询和连接的性能，并增加存储开销；
  5. 尽可能使用varchar/nvarch代理char,降低存储空间；
  6. 删除表推荐用truncat table，然后在drop table，避免表删除数据表锁时间比较长；
